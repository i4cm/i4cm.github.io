iTunes / iOS Restore Error Code Lookup

Seeing an error while restoring or updating your iPhone with iTunes or aisi-helper? Enter the code (e.g. 4013, 9, 3194, 4014) - first self-fix with Apple's official triage; if every software step fails, check the repair-level cause to tell whether it is a hardware fault.

Step 1: self-fix by the official triage (software layer)

Apple groups update/restore errors by triage direction; work them in order and most connection/network errors fix themselves. For a desktop tool with one-click driver repair and a built-in error-code helper, try aisi-helper iTunes error-code tool and one-click flash / un-brick guide; if the device won't connect to the computer at all, see iPhone won't connect to computer.

Check security software2 / 4 / 6 / 9 / 1611 / 9006
Disable or adjust third-party antivirus and firewall so they don't block your computer from reaching Apple's servers, then retry the update.
Check USB connection13 / 14 / 1600-1604 / 2000-2009 / 4005 / 4013 / 4014
Use a genuine or MFi cable and a rear USB port (not a keyboard port or hub); if it still fails, change cable, port and computer.
Check network / Apple servers17 / 1638 / 3000-3020 / 3194 / 3200
Switch networks, check the hosts file and disable the proxy; 3194 usually means a downgrade to unsigned firmware - flash the latest signed build instead.
Check hardware1 / 3 / 6 / 11 / 12 / 16 / 20-56 / 1667 / 1669
With a known-good cable, computer and network, restore twice more; if it persists it is usually a hardware fault and needs professional repair.
Jailbreak / downgrade / outdated software1015, 'The required resource can't be found.'
Jailbroken or downgrading devices trigger this, and so does an outdated iTunes - update, then use recovery mode to flash the latest iOS.
Device software too new10
The device iOS is newer than the computer's iTunes - update macOS / iTunes / Apple Devices app.

Step 2: software all failed -> repair-level diagnosis (hardware layer)

If the same code persists after two restores with a known-good cable, computer and network, it is usually hardware. Quick real-world triage

  • Stuck around 80% during restore: 80% is usually the post-write verification stage - make sure the battery is >=50% and the voltage is stable so verification doesn't fail.
  • Error 9 appears only long after unplugging: the storage isn't being read (cold joint / power), so software retries won't help - prepare for repair.
  • Error 3194 / 4013 when downgrading old iOS: the old firmware is no longer signed, or a third-party tool rewrote the hosts file to redirect gs.apple.com - restore the hosts file and flash the latest signed firmware.
  • A refurbished phone in a boot loop with 4013, restarting the moment current passes ~550 mA: the buck/charging power rail isn't working (real repair-bench diagnosis) - hardware.
  • Driver / iTunes won't install with related errors: reinstalling the Apple Mobile Device driver / AMDS via a repair toolkit is the most reliable fix.
